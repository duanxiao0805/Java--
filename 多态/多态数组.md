

比如有一个继承的结构

A->B->C

创建1个A,2个B,2个C统一放在一个数组里面,并调用他们的方法

```java
		A[] a = new A[5];
        a[0] = new A();
        a[1] = new B();
        a[2] = new B();
        a[3] = new C();
        a[4] = new C();
```



**如果想访问B C 特有的方法,怎么实现呢?**

先思考,为什么不能访问?

可以这样理解:

A把B和C生成了,那么B和C肯定有A共有的方法,所以呢,跟A重写的方法都可以访问(前提是非私有)

但是B和C特有的方法呢?

比如B有一个fb(),C有一个fc(),那么就不能直接访问了

打个比方就是,父亲生了你,就有你的特性,这个特性就不能通过父亲类创建好的你来访问

父亲 a=new 你( );  这样不能用a对象访问你的特性......

这个时候向下转型就可以了,就是把父类定义的a,强制转化为功能更多的子类!!!

````java
((B)a[1].fb())
````
